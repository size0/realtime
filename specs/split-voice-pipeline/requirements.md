# 百聆式拆分语音链路需求

## 背景

现有网页把浏览器麦克风持续接入 Qwen Omni Realtime。虽然语音自然、支持打断，但音频输入与输出持续计费，且声音、断句、识别、回答模型耦合，无法只保留喜欢的音色并替换回答模型。

本次参考百聆的模块化思路，保留现有树洞 UI、账户、后台、字幕历史和 Qwen Realtime 回退链路，新增默认的低成本拆分模式：

`浏览器 PCM → 本地 VAD → 本地 ASR → 文本模型路由 → 独立 TTS → 浏览器播放`

不直接复制百聆代码，以规避其 README 商用声明与 MIT LICENSE 的冲突。

## 用户故事

1. 作为访客或登录用户，我打开网页后可以使用低成本语音模式自然对话。
2. 作为用户，我短暂停顿时系统不应立即抢答，默认静音约 1100ms 后再提交一句话。
3. 作为用户，我在 AI 朗读时开口，当前音频应立即停止，新的讲话应优先被识别。
4. 作为用户，我希望固定使用一个温柔树洞音色，而回答内容可由其他文本大模型生成。
5. 作为运营者，我希望普通聊天走便宜模型，复杂问题才自动升级到强模型。
6. 作为运营者，当拆分语音服务异常时，用户仍可切换到现有 Qwen Realtime 模式。
7. 作为运营者，我不希望音频、密钥、原始事件或完整对话被服务器持久化。

## 功能要求

### 低成本语音模式

- 浏览器采集单声道麦克风，开启回声消除、降噪和自动增益。
- 浏览器把音频重采样为 16kHz、16-bit little-endian PCM，经受保护的 WebSocket 发送到语音 worker。
- worker 使用 Silero VAD 判断开始与结束讲话，默认：
  - `threshold=0.5`
  - `min_silence_duration_ms=1100`
  - `speech_pad_ms=160`
- worker 使用本机 SenseVoice/FunASR 完成中英文自动识别。
- ASR 完成后，前端把文字及最近对话历史提交到现有回答 API。
- 回答 API根据问题复杂度在普通模型和强模型之间路由，并在响应中返回实际使用的路由层级和模型名。
- 前端把完整回答按自然句子切分，逐句提交到独立 TTS。
- TTS 默认使用 Qwen3 TTS Realtime 的固定 `Cherry` 音色、24kHz 单声道 PCM、客户端 `commit` 模式。
- 浏览器边收到 PCM 边播放，不保存音频。

### 插话与状态

- worker 检测到 `speech_started` 时立即通知浏览器。
- 浏览器收到 `speech_started` 后：
  - 停止并释放所有排队或正在播放的 TTS 音频；
  - 取消尚未完成的回答请求；
  - 通知 worker 清空尚未提交的 TTS 文本；
  - 把正在朗读的助手字幕标记为 `interrupted`。
- 现有空闲、请求权限、连接中、聆听、思考、说话、静音、断开、错误状态继续使用。

### 回退模式

- 页面默认选择低成本拆分模式。
- 用户可在通话前切换到“高保真 Qwen Realtime”回退模式。
- 低成本 worker 无法连接或模型未就绪时，页面显示明确中文错误，并提供切换回退模式的入口。
- 回退模式保留现有声线列表和 WebRTC 实现。

### 隐私与存储

- 最近 200 条字幕仍只保存在浏览器 localStorage。
- 服务器不保存音频、TTS PCM、原始 WebSocket 事件和完整提示词/对话。
- 后台仍只记录连接次数与模型回复次数，不新增对话内容查看。

## 安全要求

- 新增 `POST /api/voice/token`，必须同时通过：
  - 有效登录会话；
  - 同源校验；
  - CSRF 校验；
  - IP + 用户连接令牌频率限制；
  - 用户每日连接额度。
- 令牌为 HMAC-SHA256 签名、短期有效、含版本、用户 ID、过期时间和随机 nonce。
- WebSocket worker 验证签名、过期时间、Origin 白名单、音频块上限、JSON 字段类型和文本长度。
- 同一用户默认只允许一个 worker 会话；单次会话最长 30 分钟。
- API Key 和签名密钥仅从服务端环境变量读取。
- 日志只记录事件类型、耗时、错误码和匿名会话 ID，不记录音频、密钥、令牌、问题或回答正文。
- 错误响应不得返回上游原始响应、栈信息或配置值。

## 验收标准

- 用户连续中英文说话可得到本地转写、文本回答和流式音频朗读。
- 约 0.5 秒停顿不会提交，约 1.1 秒静音后提交。
- AI 朗读时讲话可在 250ms 内停止本地播放。
- 普通树洞聊天返回 `economy` 路由；明确复杂问题返回 `strong` 路由。
- worker 未运行、ASR 未就绪、TTS 失败时有明确中文提示且不会泄漏密钥。
- 刷新后文字字幕恢复，音频不恢复；清空字幕后 localStorage 清除。
- 类型检查、Lint、Vitest、Python 单元测试、生产构建全部通过。
